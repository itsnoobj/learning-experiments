# A Field Guide to Being Human

> The goal isn't to learn 200 mental models. The goal is to become someone with better judgment.

## Intent

A personal operating system for navigating workplace dynamics, relationships, difficult people, and hard decisions — organized by **situation**, not by book or concept.

Not a book you finish once, but a reference you return to when you face:

- A manipulative coworker
- An unfair promotion
- A difficult manager
- A conflict in a relationship
- A high-stakes decision
- A crisis of confidence
- A question about what's right versus what's effective

## Philosophy

Every difficult situation in work or life is usually a combination of a handful of recurring forces:

| Force | What it drives |
|-------|---------------|
| Incentives | Why people do what they do (not what they say) |
| Ego | Why admitting fault is so hard |
| Fear | Why people avoid, deflect, and freeze |
| Trust | Why collaboration works or breaks |
| Status | Why politics exists everywhere |
| Identity | Why people defend bad positions |
| Scarcity | Why people hoard, compete, and protect |
| Power | Why hierarchies distort behavior |
| Uncertainty | Why people cling to control |
| Reciprocity | Why favors create invisible debts |

If you understand these forces, you can understand almost any situation.

## Chapter Template

Every topic follows the same structure for consistency and retrievability:

1. **Question** — A real-world dilemma (e.g., "Why do people resist change even when it benefits them?")
2. **Workplace Scenario** — A realistic example you might encounter
3. **Core Human Dynamic** — Which recurring force is driving the behavior
4. **Stories** — One from an epic/philosophy, one from history/biography, one from business/everyday life
5. **Psychology** — Cognitive biases, emotions, or social dynamics involved
6. **Mental Models** — Reusable concepts that explain the situation
7. **Trade-offs** — When following the principle too rigidly causes problems
8. **Practical Framework** — A checklist or decision process you can use
9. **Reflection** — Questions to apply it to your own context
10. **Connections** — Links to related topics

### Multi-Lens Analysis

Each chapter also answers: *how would different thinkers approach this?*

| Thinker | Lens |
|---------|------|
| Krishna | Context, duty, long-term outcomes |
| Chanakya | Statecraft, incentives, pragmatism |
| Marcus Aurelius | Self-mastery, virtue, emotional control |
| Charlie Munger | Mental models, incentives, rationality |
| Abraham Lincoln | Empathy, coalition building, humility |

Not because they're always right, but because each represents a distinct philosophy. The comparison teaches that there is often more than one reasonable way to approach the same problem, depending on your values and constraints.

## Sources

Stories and wisdom drawn from:

- **Epics**: Mahabharata, Ramayana, Bhagavad Gita, Greek mythology, Panchatantra
- **Philosophy**: Stoicism (Meditations), Arthashastra, Tao Te Ching
- **Biographies**: Lincoln, Mandela, Jobs, Kalam, Munger, Frankl
- **Psychology**: Kahneman, Cialdini, Frankl, Ariely
- **Business**: Grove (Only the Paranoid Survive), Bezos (letters), Dalio (Principles)

## Approach

1. **Build iteratively** — Start with chapters that match real, current problems
2. **Situation-first** — Always start from "I'm facing X" not "Let me study Y"
3. **Stories over lectures** — Every principle anchored in narrative
4. **Tension over answers** — Especially in ethics; judgment grows from sitting with dilemmas
5. **Practical over academic** — Every chapter ends with something you can do

## Creating New Stories — Step-by-Step Workflow

This section documents exactly how to create new content from scratch: story → quiz → visual → audio → live in the app.

### Prerequisites

```bash
# From project root
npm install           # monorepo deps (runs turbo)
cd tools/visuals && npm install   # rough.js + jsdom for SVG generation
cd tools && pip install -r requirements.txt  # kokoro TTS for audio (optional)
```

### Overview: What One Chapter Needs

```
content/chapters/part-{NN}/{ID}.json       ← Story content (required)
content/chapters/part-{NN}/{ID}.quiz.json   ← Quiz challenges (required)
content/chapters/part-{NN}/{ID}.svg         ← Visual illustration (required)
apps/web/public/content/{ID}.svg           ← Copy of SVG for web serving
apps/web/public/content/{ID}.mp3           ← Audio podcast (optional)
```

Everything flows from the chapter JSON. The quiz tests understanding of the chapter. The visual depicts the story scene. The audio reads the chapter aloud.

---

### Step 1: Pick a Chapter from OUTLINE.md

Each chapter in `OUTLINE.md` has a number, question, core force, and suggested story source.

Example: Chapter 4 — "Why does praise become addictive?" (Identity force, Panchatantra monkey & crocodile)

The chapter number IS the file ID. Part I chapters are 1-25, Part II chapters are 26-48, etc.

---

### Step 2: Write the Chapter JSON

Create `content/chapters/part-{NN}/{ID}.json`:

```json
{
  "id": "4",
  "part": 1,
  "section": "A",
  "title": "Why Does Praise Become Addictive?",
  "forces": ["identity", "incentives"],
  "connections": ["3", "5", "16", "39"],
  "audio": "4.mp3",
  "visual": "4.svg",
  "sections": [
    { "type": "situation", "content": "..." },
    { "type": "story", "title": "The Story", "content": "..." },
    { "type": "contrast", "title": "The Contrast", "content": "..." },
    { "type": "principle", "content": "..." },
    { "type": "psychology", "title": "Why It Happens", "content": "..." },
    { "type": "trap", "title": "The Trap", "content": "..." },
    { "type": "move", "title": "The Move", "content": "..." }
  ]
}
```

Section writing guidelines:

| Section | Length | Voice | Key Rule |
|---------|--------|-------|----------|
| situation | 2-4 sentences | 2nd person ("You're in a meeting...") | Instantly relatable |
| story | 300-500 words | 3rd person narrative | One clear moment of decision |
| contrast | 100-150 words | 3rd person | Different tradition, same principle |
| principle | 2-3 sentences | Direct | Quotable, named concept |
| psychology | 150-250 words | Explanatory | Name biases, explain WHY brain does this |
| trap | 100-150 words | Cautionary | What breaks if overdone OR ignored |
| move | Numbered steps | Actionable | What to do THIS WEEK |

Story Quality Rules:
1. No logic gaps — every cause→effect obvious
2. Simple beats clever — instant visual
3. Visceral over intellectual — gut before head
4. No translation needed — introduce all characters/sources
5. The lesson is obvious without explanation
6. One clear moment — zoom into the frame

Full template details: `CHAPTER_TEMPLATE.md`

---

### Step 3: Generate the Quiz JSON

**Option A: Use the prompt generator**

```bash
node tools/generate-quiz.mjs content/chapters/part-01/4.json
```

This generates a `4.quiz-prompt.txt` file containing a ready-to-paste LLM prompt. Copy the prompt into Claude/ChatGPT and save the output as `4.quiz.json`.

**Option B: Write manually**

Create `content/chapters/part-{NN}/{ID}.quiz.json`:

```json
{
  "chapterId": "4",
  "challenges": [
    { "type": "scenario-choice", ... },
    { "type": "spot-the-force", ... },
    { "type": "card-flip", ... },
    { "type": "before-after", ... }
  ],
  "principle": {
    "text": "Core principle in one sentence",
    "subtext": "Supporting explanation (1-2 sentences)"
  },
  "reflection": "A personal question with no right answer"
}
```

Rules:
- Use at least 3 different template types per quiz
- `scenario-choice` should always be first (warm-up)
- 3-5 challenges per quiz
- Feedback should TEACH, not just say "wrong"
- Test understanding, not memorization

Available templates: `scenario-choice`, `spot-the-force`, `card-flip`, `drag-match`, `before-after`

Full template schemas and examples: `CREATING_TEMPLATES.md`

### Step 4: Generate the SVG Visual

Each chapter gets one hand-drawn illustration depicting the story scene.

**Add a scene to `tools/visuals/generate.mjs`** — find the `SCENES` map and add an entry:

```javascript
'4': {
  part: '01',
  quote: '"Praise is a fine reward and a terrible master."',
  draw(ctx) {
    const { line, circle, rect, path, poly, ellipse, text, rc, svg, ink } = ctx;
    const gold = '#DAA520'; // accent color for this scene

    // --- Draw your scene here using the helpers ---
    // line(x1, y1, x2, y2, opts)
    // circle(cx, cy, diameter, opts)
    // rect(x, y, w, h, opts)
    // path('M x y Q ...', opts)
    // poly([[x,y], ...], opts)
    // text('label', x, y, { size, fill, italic })

    // Example: stick figure
    circle(400, 200, 30);              // head
    line(400, 215, 400, 300);          // body
    line(400, 300, 380, 360);          // leg
    line(400, 300, 420, 360);          // leg
    line(400, 240, 360, 280);          // arm
    line(400, 240, 440, 280);          // arm
  },
},
```

**Run it:**

```bash
cd tools/visuals
node generate.mjs 4           # Generate one chapter
node generate.mjs 1 2 3 4 5   # Generate multiple
node generate.mjs --all        # Generate all registered scenes
```

Visual Style Rules (from `VISUAL_STYLE.md`):
- **800×600** white background
- **Stick figures** — circle head, line body, V-legs
- **Black + ONE accent color** on the key object only:
  - Gold `#DAA520` — goals, prizes, aspirations
  - Muted red `#CC3333` — threats, danger
  - Soft blue `#4A90D9` — clarity, insight
  - Green `#5B8C5A` — growth, safety
- **Hand-drawn wobble** — roughness ~0.8-1.5
- **Scene depicts the story** — not an abstract concept
- **One italic quote** at bottom — the principle or punchline
- **No** speech bubbles, labels, gradients, or detailed faces
- **Body posture tells the story** — arms up=reaching, crossed=refusing

Composition rules:
- Top = goal/aspiration, Bottom = ground/status quo
- Characters big enough for body language to be readable
- Every line earns its place — remove anything that doesn't help tell the story

### Step 5: Generate Audio (Optional)

The audio script reads a markdown version of the chapter and creates a podcast-style MP3 with two voices.

```bash
cd tools
python chapter_to_podcast.py ../chapters/part-01/1-why-do-i-defend-wrong-decisions.md
```

Voice assignments:
| Role | Voice | Used For |
|------|-------|----------|
| Narrator | `af_bella` | Situation, Principle, Psychology, Trap, Move |
| Storyteller | `am_adam` | The Story, The Contrast |

Options:
```bash
# Custom output path
python chapter_to_podcast.py chapter.md -o output/podcast.mp3

# Different voices
python chapter_to_podcast.py chapter.md --narrator af_heart --storyteller am_michael

# List available voices
python chapter_to_podcast.py --list-voices
```

Requirements: Kokoro-82M model (downloads ~300MB on first run, then fully offline). See `AUDIO_APPROACH.md` for details.

Note: The script reads markdown. If your chapter is JSON-only, write a brief markdown version or adapt the script to read from JSON sections.

### Step 8: See It Live

The app uses dynamic routing. Once the JSON files exist and assets are in `public/content/`, your chapter is live:

- **Chapter page:** `/chapter/{ID}` (e.g., `/chapter/4`)
- **Quiz page:** `/quiz/{ID}` (e.g., `/quiz/4`)

No code changes needed — the dynamic route scans `content/chapters/part-*/` for matching IDs.

### How the Map Auto-Generates

The map is driven by `content/hierarchy.json`. Each world has regions, and each region has a `missions` array of chapter IDs:

```json
{
  "id": "A",
  "title": "Identity",
  "missions": ["1", "2", "3", "4", "5", "6", "7", "8"]
}
```

When you add a chapter whose ID is already listed in a region's `missions` array, it automatically appears as a clickable node on the mission map. The map reads the chapter's title from its JSON to display on hover.

To add NEW missions to an existing region, edit `content/hierarchy.json` and add the ID to the region's `missions` array.

To add a new region or world, add the full object following the existing structure (see `HIERARCHY.md` for details).
